# 📚 SignBell 프로젝트 최종 회고록

### 📌 프로젝트 개요
- **프로젝트 명**: SignBell (사인벨)
- **프로젝트 목표**: WebRTC와 AI를 활용한 실시간 수어 학습 및 게이미피케이션 플랫폼
- **개발 기간**: 2025-10-14 ~ 2025-11-01 (약 3주)
- **나의 역할**: UI/UX 총괄, 프론트엔드 개발, 및 인프라 구축 (Docker, Kubernetes, Jenkins CI/CD)

<br>

### 🚀 주요 성장 과정

**1주차 (10/14 ~ 10/17): 기획 및 UI/UX 설계**
프로젝트 첫 주, '실시간 AI 수어 퀴즈'라는 핵심 아이디어를 구체화하는 데 집중했습니다. 사용자 흐름(User Flow)을 정의하고, '다중 탭 접속'과 같은 예외 정책을 기획 단계에서부터 논의하고 확정했습니다.
이를 바탕으로 와이어프레임, 컬러 가이드, 핵심 컴포넌트 디자인을 완료했습니다. 명확한 시각적 목표를 설정한 덕분에, 팀 전체가 일관된 방향으로 개발을 시작할 수 있는 기반이 마련되었습니다.

**2주차 (10/20 ~ 10/24): 핵심 UI 구현 및 기능 고도화**
설계안을 바탕으로 본격적인 프론트엔드 개발에 착수했습니다. 저는 **사용자 인증, 온보딩, 랜딩 페이지, 랭킹 UI** 등 서비스의 핵심이 되는 페이지들을 구현했습니다.
특히, 가장 복잡도가 높았던 '실시간 퀴즈 대기방/게임방' UI를 구현하며, WebSocket 연동 전에도 테스트할 수 있도록 동적 로직(선착순 도전, 상태 관리)과 테스트 버튼을 미리 구현했습니다.  이는 이후 백엔드 및 WebRTC 연동 시 병목 현상을 줄이고 병렬적인 개발을 가능하게 했습니다.

**3주차 (10/27 ~ 11/01): 인프라 구축 및 CI/CD 완성**
프로젝트의 마지막 주는, 개발된 기능들을 실제 서비스로 배포하는 데 집중했습니다. 저는 **Docker를 사용해 프론트엔드/백엔드/FastAPI 서버를 각각 컨테이너화**하는 작업을 진행했습니다.
나아가, **Kubernetes(K8s) 클러스터를 구축**하여 이 컨테이너들을 안정적으로 오케스트레이션하고, **Jenkins를 활용해 CI/CD 파이프라인을 완성**했습니다. Git push(병합) 시 자동으로 빌드, Docker 이미지 생성, Kubernetes 배포가 이루어지는 자동화 시스템을 구축하며 프로젝트를 마무리했습니다.

<br>

### ✒️ 최종 회고

**👍 좋았던 점**
- **설계 기반의 신속한 개발**: 초기에 UI/UX 설계를 명확히 확정한 덕분에, 개발 단계에서는 디자인 고민 없이 기능 구현에만 집중할 수 있었고, 일관된 사용자 경험을 제공할 수 있었습니다.
- **프론트엔드의 선제적 로직 구현**: 퀴즈방의 복잡한 상태 관리를 백엔드 연동 전에 미리 시뮬레이션하고 구현한 것이 효과적이었습니다. 이는 팀 전체의 개발 속도를 높이는 데 기여했습니다.
- **개발부터 배포까지의 End-to-End 경험**: UI/UX 설계부터 시작해, React 개발, 그리고 Docker/Kubernetes를 이용한 인프라 구축 및 CI/CD 자동화까지, 서비스의 탄생과 운영 전 과정을 총괄해 본 것은 귀중한 자산이 되었습니다. 

**🤔 배운 점**
- **UI/UX의 범위**: '다중 탭 접속 제한' 정책을 논의하며, UI/UX가 단순히 시각적인 요소를 넘어 서비스의 정책과 기술적 제약을 사용자에게 전달하는 중요한 역할임을 다시금 깨달았습니다.
- **인프라와 상태 관리의 연관성**: 퀴즈방에서 '새로고침(F5)' 시 발생하는 상태 동기화 이슈를 통해, 프론트엔드의 상태 관리가 백엔드 로직은 물론, WebSocket 세션과 같은 인프라 수준의 동작 방식과도 긴밀하게 연결되어야 함을 배웠습니다.
- **CI/CD의 필요성**: 자동화된 배포 파이프라인을 구축하면서, 반복적인 배포 작업을 자동화하는 것이 왜 개발 효율성과 서비스 안정성을 높이는 데 필수적인지 체감할 수 있었습니다.

<br>

이번 프로젝트는 UI/UX 설계라는 '청사진'을 그리는 역할과, CI/CD라는 '자동화 공장'을 짓는 역할을 동시에 수행하며, 아이디어가 실제 서비스로 배포되기까지의 전 과정을 깊이 있게 경험할 수 있었던 의미 있는 시간이었습니다.
